Please read first!

In order to run our website, please navigate into the site/ directory and run the 'start.sh' bash script. This script both intialises the Mongo database connection, and starts the server. You may need to edit the Node.js command to either 'node' or 'nodejs' depending on your operating system. Then navigate to 'localhost:8081' to view our website.
We excluded any node_modules and existing data to reduce our submission size. Use 'npm install' to install our dependencies, listed in package.json.

Client Side Report

We made good use of Google to find resources covering virtually every element of our website. We found many useful demos on Stack Overflow and W3 Schools, which we read and adapted for certain features. All HTML pages are valid HTML 5 Polyglot, even with server side implemented, tested using Total Validator.

General

  • Navigation bar uses only HTML and some advanced CSS. This was first taken from a codepen demo, however it originally used FontAwesome for the icons. This has been adapted so it now uses custom made SVGs which were all made in Inkscape. The only exception being the cutlery icon. The CSS for the nav bar now looks much tidier and the styling such as colours and positioning has been changed. This nav bar took a long time and a lot of work to perfect, but it looks good now it's done.
  • The whole site has fluid design, remaining completely useable no matter what window size it is being viewed in. We have not however tested the site on mobile.
  • Bitmap Graphics
    • CompleatFitness logo created from scratch using Gimp and royalty-free font 'Pecita'
    • favicon created from scratch using Gimp and royalty-free font 'Pecita'
  • Vector Graphics
    • Home Icon - Made with Inkscape, this icon required me to draw different shapes like circles, triangles and squares. I also needed to use the Path tools a lot which enabled me to draw a shape over another shape and take the difference of the two shapes. This is the method I used to create the door opening in the house.
    • Barbell Icon - This icon was made by drawing a circle within another circle and finding the difference of the two shapes to create the circle border. I then just used the rectangle shape for the bar, and used rounded rectangles for the weights.
    • Gallery Icon - This icon was made by finding the difference of two rectangles for the border. I then just used triangles and a circle for the image.
    • Contact Us Icon - This was made by finding the difference of two circles for the border. I then proceeded to draw a rounded rectangle and then overlayed two rectangles where the white cut outs are. I used the 'group items' feature on these two rectangles to make a single shape and then found the difference between this shape and teh rounded rectangle I drew earlier to create the envelope shape.

index.html:

  • Multiple different background images were tried and tested against our theme's colours. Gimp was used to make copies with varying levels of opacity. The end result was mostly transparent (10% opacity), improving the legibility of overlying text.

packages.html:

  • Learned to write accordion from inspirationalpixels. It was then styled to our needs and works very well for FAQs. The design is modular so it is very easy to add a new item in to the accordion.
  • The BMI Calculator uses JavaScript to take values from weight and height sliders, it then calculates BMI, and replaces the innerHTML of a DOM to display the result. This happens on each change of sliders thanks to jQuery.

nutrition.html:

  • The comment box features a text input attribute for the name of the commentor and a text area input attribute for their comment. AngularJS is used to change the internal text of a paragraph tag when the user needs to be alerted of an empty field etc.
  • We used the 'white-space : pre-line' value in commentbox.css to ensure that line breaks entered in the text area were displayed once the comment had been posted. We also gave each table row a maximum height and enabled scrolling within comments for an improved aesthetic outcome.

gallery.html:

  • For image upload, a simple form attribute was used. The button is formatted with the signature green colour of our website, and the file input button is left unedited.
  • The images are displayed in image tags, and CSS is used to give them all a maximum height of 300px. As mentioned in the server side report, we would ideally use a slider, or a layout that would work dynamically on different screen sizes.
  • Instafeed is used to add pictures from Instagram using only JavaScript. An Instagram developer account was needed to register the application before pictures could be accessed.
  • A more difficult task was to put the Instagram pictures in to a nice image slider called Slick. This now works well and looks nice since the images are larger and scroll automatically.

contact.html:

  • Contact form elements were looked up on W3Schools and styling from different sources searching online to see what could look good on our site.
  • Google map was implemented by following the instructions taken from the Google Map API intro page.
